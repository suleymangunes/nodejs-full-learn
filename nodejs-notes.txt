Nodejs nedir?
Node.js bir javascript çalışma ortamıdır. 

Nodejs nasıl çalışır?
Event-driven: Olay odaklı çalışır. Tek thread kullanmasından ötürü istenilen işleri bir olay döngüsünde değerlendirir. 
Event driven olay temelli yaklasimda programin akisini olaylar belirler. olay dongusu icerisinde tamamlanan islemler callbak 
vasitasiyla islemin tamamlandigini bildirir. 
Non-blocking: Engellenmeyen olarak çalışır. Node.js işlem sırasına koyduğu bir olayın tamamlanmasını beklemeden diğer olayı işleme alabilir. Bu şekilde iş akışı engellenmez. 
Bloklanmayan bir yazilimda kodun bir parcasinin calismasi durumunda kodun diger bolumunun calismamasi durumudur.
Bu sayede kodun akisini engelleyecek surede uzun islem sonucu bekleyen fonksiyonlar beklerken geri kalan kisimlar calisir yani bloklanmaz.
İslem sonucu surenin sonunda belirlenen fonksiyon da sonucunu da yansitir.
Asynchronous: Asenkron çalışır. Asenkron çalışma kod akışının yukardan aşağıya ilerlemediği işlemlerin birbirini beklemediği çalışma türüdür. Bu sayede işlem sırası olaya göre belirlenebilir. 

Process Object
Process nesnesi nodejs çalışma ortamında o an yapılan işlerle ilgili bilgi içerir. 

REPL Read-Eval-Print-Loop
Repl terminalden interaktif bir şekilde JavaScript çalıştırarak sonuçları görmeye yarar. Debug, test gibi denemeler için kullanılabilir. 

Asenkron Nodejs
Nodejs asenkron bir şekilde çalışır. Senkron sirayla is yapar ve bir sonraki is icin onceki isin bitmesini bekler.
Asenkron ise bir isi yapmaya basladiktan hemen sonra diger ise baslar bu sayede islerin birbirini bekleme zorunlulugu ortadan kalkar.
Bu sayede mümkün oldugunsa kisa surede en cok is yapilir.

V8 Javascript Engine
V8 engine ile kodlar makine koduna cevrilir.

Libuv
Nodejs kodlarinin asenkron calismasini saglayan yapidir.

Modül 
Nodejs uygulamasi farkli gorevleri olan farkli javascript dosyalarindan olusur ve nodejs iceridigi tum javascript dosyalarina 
bir modul gibi davranir. Modul genelde belirli ozel bir islevi olan javascript dosyasidir.
Bu sekilde nodejs uygulamaya ait olan dosyalari farkli bolumlere ayirarak kodun daha modularize olmasini ayni zamanda bu kod kontrolunun
ve hata yakalamanin daha kolay olmasini saglar.

NPM
Node package manager 3. kisiler tarafindan yazilan paketlerin bulunabilecegi repo ortami sunar.

Package
Paket genelde birden fazla modulden olusan ortak islevi olan yazilim parcalaridir.
Tum bu kodlari yonetmek icin bilgileri tasiyan dosya package.json'dir.

Request Response Loop
Nodejs ile yazilan bir web uygulamasi temelde bir request-response dongusudur.
Tarayiciya www.google.com yazmakla aslinda uzaktaki sunucuya bir request (istek) gonderilir.
Uzaktaki sunucu ise Google sayfasini gostermek icin response (cevap) doner.

IP Adsress
İstekte bulunan sunucuya IP adresi ile ulasilir. Aslinda www.google.com DNS ile ip adresine alan adi ile ulasilmasini saglar.
DNS sucunusu yardimi ile ip adresine ulasir ve chunk adli parcalar ile karsi sunucudaki bilgiler alinir.
Sunucu uzerinden istenilen yerden degil uygun port ile haberlesme saglanir.

TCP - IP Protokolu
Istemci ile sunucu arasindak baglanti TCP - IP protokolu ile kurulur. TCP istemci sunucu arasinda veri iletisimi
saglayan protokollerden olusan yapidir. Iletisim TCP - IP ile kuruldu.

HTTP
Kurulan iletisimden sonra istemci ve sunucu arasinda anlasma da http ile saglanir. Sunucu ile istemci arasinda
bazi kurallar vardir. Bu kurallari http saglar. HTTPS guvenlik eklenmis http'dir.

Calisma Ortami
Gerekli calisma ortami kurulmadan once githubda repository olusturulur. Calisma ortami repository ile iliskilendirilir.
Npm paketlerini kullanmak icin npm initialize edilir. Projede kullanilacak sifre gibi ozel kullanimlarin githubda paylasilmamasi
icin .gitignore dosyasi olusturulur ve ozel kisimlar bu kisimda tanimlanir.

Dependencies Dev-Dependencies
Dependencies projenin calismasinda kullanilan kutuphaneleri kapsar. Dev-Dependencies ise proje gelistirilirken kullanilan kutuphaneleri kapsar.
Ornegin nodemon ile kodda her degisiklik yapilinca sunucu bastan calistirilmalidir ancak nodemon ile bu sorun cozulur.
Projenin calismasinde degil sadece gelistirilmesi surecinde kullanildigi icin dev kisminda olmalidir.

Statik Dosyalar
Bir web uygulamasi olustururken bu uygulamanin iskeleti statik dosyalardir. Olusturulmak istenen uygulamanin iskelet yapisi css, html,
js, dosya ve gorselleri gonderilerek olur. Bir express uygulamasinda statik dosyalarin gonderilmesi icin express.static gomulu middleware fonksiyonu kullanilir.

Middleware 
Nodejs uygulamasi temelde bir request response dongusudur. İstege karsi bir cevap alinir.
Bu istek-cevap dongusunun icerisindeki gorevi olan her fonksiyon middleware fonksiyonudur.
Middleware kullanarak her hangi bir fonksiyon calistirilabilir. Request ve response nesneleri degistirilebilir.
Request response dongusu sonlandirilabilir. next ile sonraki middleware cagrilabilir.
get requestte bir middlwaredir ancak burda next ile degil send ile de sonlandirilir

Template Engine
Degiskenlerin statik degil dinamik olmasi tercih edilir. Bunun icin Template engine kullanilir.
Bu sayede bir static dosya ile dinmaik icerik beraber kullanilabilir.
